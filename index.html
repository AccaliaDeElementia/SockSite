Contributing

We generally use the following branch system for contributing:

  • For development, use a local fork. You can quickly launch a dev server using Vagrant from the root of the project
  • Submit your pull request against socksite/testing, so it can be integrated with other in-flight changes in our demo environment
  • When it's all good, it'll be pushed to master and deployed!
